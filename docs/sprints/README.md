# Sprint Documentation

Use this folder for pre-implementation sprint reviews and post-sprint summaries.

## Workflow standard
1. Write a sprint review doc **before coding**.
2. Include objective, scope, acceptance criteria, TDD plan, risks, and go/no-go gates.
3. Start implementation only after explicit sign-off.
4. After implementation, append a concise sprint outcome note.
